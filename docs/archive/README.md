# Documentation Archive

This directory holds development history that is useful for traceability but should not sit in the active docs index.

## Buckets

- `handoffs/`: session handoffs, restart prompts, and milestone transfer notes.
- `reviews/`: review outputs, per-file review notes, and audit findings.
- `planning/`: older project-state, goal, monolithic planning, and long-form design notes.
- `audits/`: larger audit workstreams that were previously top-level work areas.

## Rule of Thumb

Keep current user-facing documentation in `docs/`, `docs/spec/`, and `docs/steps/`.
Put dated notes, one-off restart files, generated review output, and completed workstream state here.

Historical files may still mention their original paths. That is expected; archive contents are preserved as records, not normalized as current instructions.
